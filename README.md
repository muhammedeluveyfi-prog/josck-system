# نظام إدارة العمليات - JOSCK System

نظام متكامل لإدارة عمليات حركة الأجهزة داخل القسم من لحظة دخول الجهاز حتى تسليمه للزبون.

## المميزات

### واجهة موظف إدارة العمليات
- إضافة أجهزة جديدة مع جميع المعلومات المطلوبة
- عرض الأجهزة قيد التصليح والمكتملة
- تحويل الأجهزة للفنيين أو انتظار الموافقة
- إحصائيات شاملة للأجهزة
- عرض أيقونات الفنيين مع عدد الأجهزة لكل فني
- إدارة الأجهزة الجديدة والمستلمة من الفني

### واجهة الفني
- عرض الأجهزة المحولة التي يجب استلامها
- عرض الأجهزة قيد التصليح
- إضافة تقارير على الأجهزة
- تحويل الأجهزة إلى موظف العمليات بعد الانتهاء

### واجهة الاستعلام (خدمة العملاء)
- البحث عن جهاز برقم الطلب أو رقم الهاتف
- عرض حالة الجهاز وتفاصيله
- عرض جميع التقارير المضافة على الجهاز
- معرفة موقع الجهاز الحالي

### واجهة المدير
- إدارة المستخدمين (إضافة، تعديل، حذف)
- تعيين الصلاحيات للمستخدمين
- عرض إحصائيات شاملة للنظام
- تقارير عن الأجهزة الأخيرة

## التقنيات المستخدمة

- **React 18** - مكتبة واجهة المستخدم
- **TypeScript** - للبرمجة الآمنة
- **Vite** - أداة البناء
- **React Router** - للتنقل بين الصفحات
- **date-fns** - لإدارة التاريخ والوقت
- **lucide-react** - للأيقونات
- **LocalStorage** - لتخزين البيانات محلياً

## التثبيت والتشغيل

### المتطلبات
- Node.js 18 أو أحدث
- npm أو yarn

### خطوات التثبيت

1. تثبيت المكتبات:
```bash
npm install
```

2. تشغيل المشروع في وضع التطوير:
```bash
npm run dev
```

3. بناء المشروع للإنتاج:
```bash
npm run build
```

## حالات الجهاز

- `new`: جهاز جديد تم إضافته
- `in_repair`: جهاز قيد التصليح
- `completed`: جهاز مكتمل (تم التصليح أو لا يصلح)
- `transferred`: جهاز محول للفني
- `awaiting_approval`: جهاز في انتظار موافقة الزبون
- `received_from_technician`: جهاز مستلم من الفني

## ملاحظات مهمة

- البيانات تُخزن محلياً في LocalStorage
- لتطبيق حقيقي، يجب استبدال LocalStorage بقاعدة بيانات حقيقية
- يمكن إضافة المزيد من الميزات مثل التصدير والطباعة
